GQMaps-Manage your data

Pre-requisites

  • Get a list of IP assets.(IP Address you want to scan)
  • Enable SNMP V2 or if not supported enable V1 and get the community string from the system admin.
  • Please make sure you want to use java version 6 on the macine where you want to run the meter.
  • To run the meter please type the following command: java -jar gqmeter.jar <path to input asset file>
  • Please follow the steps described below to create the input asset file.

Asset File Creation

Hover over the lines below to view the detailed meaning.

GQMaps is smart enough to discover the type of asset given an IP Address.

Entries above the $$ mark the header section. Please choose appropriate switches as required.

Lines starting with # are comments.

Header section 

$meterid gqmaps1
@computer     snap_shot|conn_devices|process|inst_sw
@printer      snap_shot|conn_devices
@nsrg         snap_shot|conn_devices
@storage      snap_shot
@speedtest on rem_ipaddress  rem_comm_str  loc_ipaddress  loc_comm_str
$$

Input section


gqmaps 192.168.1.14
gqmaps 192.168.8.128  192.168.8.140

Samples

Sample1 - File with all switches used 

            	$meterid gqmaps1 
            	@computer     snap_shot|conn_devices|process|inst_sw 
            	@printer      snap_shot|conn_devices 
            	@nsrg         snap_shot|conn_devices 
            	@storage      snap_shot|conn_devices|process|inst_sw 
            	@speedtest	  on     192.168.8.15    gqmaps		127.0.0.1	gqm3%
            	$$
            	gqmaps 192.168.1.14 
            	gqmaps 192.168.8.128  192.168.8.130

Sample2 - File with required switches only 

            	$meterid gqmaps1 
            	@computer     process|inst_sw 
            	@printer      snap_shot 
            	@nsrg         conn_devices 
            	@storage      snap_shot
            	@speedtest	  off
            	$$ 
            	gqmaps 192.168.1.14 
            	gqmaps 192.168.8.128  192.168.8.130
